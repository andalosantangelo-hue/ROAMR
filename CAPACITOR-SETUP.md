# ROAMR — Native iOS & Android (Capacitor)

The same React app is wrapped with Capacitor to ship to the App Store and Play Store.

- App ID: `com.roamr.app`
- App name: `ROAMR`
- Web build dir: `dist`

## One-time setup (run locally — needs native toolchains)

Requirements: Node, Xcode (iOS, macOS only), Android Studio (Android).

```bash
npm install                 # installs deps incl. Capacitor
npm run build               # produces dist/
npx cap add ios             # creates native ios/ project (macOS + Xcode)
npx cap add android         # creates native android/ project (Android Studio)
```

## Day-to-day

```bash
npm run cap:ios       # build web + sync + open Xcode
npm run cap:android   # build web + sync + open Android Studio
```

Then press Run in Xcode / Android Studio to launch on a simulator or device.
For TestFlight: archive in Xcode → upload to App Store Connect.

## Native features wired-ready
- `@capacitor/camera` is installed — the Create Tribe photo picker will use the
  native camera/gallery on device (and falls back to the web file input in the browser).

## Notes
- `ios/` and `android/` native folders are generated by `cap add` and are git-ignored
  by default; run the add commands on your machine.
- Re-run `npm run cap:sync` after any web change before building natively.

## Native chrome note (do during the native build)

The web build draws a *simulated* status bar (`<StatusBar />`) and a home-indicator
bar (in `Splash.jsx`) so the browser demo looks like a phone. On a real iOS/Android
build the OS draws those itself, so they will double up.

When you build natively, hide the simulated chrome on native, e.g.:

```js
import { Capacitor } from "@capacitor/core";
const isNative = Capacitor.isNativePlatform();
// render <StatusBar /> and the home-indicator bar only when !isNative,
// and use @capacitor/status-bar + safe-area insets for the real device.
```

Easiest: gate `StatusBar` (and the `Splash` home-indicator `<div>`) behind `!isNative`,
then add safe-area padding (`env(safe-area-inset-top/bottom)`) so content clears the
real notch / home indicator. Verify on a device or simulator — that is why this is left
for the native build rather than changed blind.
